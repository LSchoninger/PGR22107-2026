# UC01: Jogar

## Descrição Geral
Este é o detalhamento do caso de uso "Jogar" do programa Hangman. Ele faz o detalhamento da lógica principal do jogo.

### Requisitos
- RF01: Jogar o jogo da Forca

### Atores
- Player

### Prioridade
 - Alta

### Pré-condições

O jogo deve ser inicializado e ter um conjunto de palavras (dicionário) que serão sorteadas

### Frequência de uso

Alta

### Condição de entrada

O usuário solicita o início de uma nova partida

## Fluxo Principal

1. O sistema seleciona uma palavra de um dicionário de palavras;
2. É apresentado ao jogador o número de letras que essa palavra possui e uma dica sobre a palavra sorteada;
3. O jogador informa um palpite de letra que a palavra possui;
4. O sistema verifica se a letra informada está presente na palavra;
   
    4.1 Caso a letra esteja presente, o sistema apresenta a palavra com as letras já descobertas visíveis;
   
    4.2 Caso a letra não esteja presente, o número de erros é incrementado;

5. O fluxo retorna ao item 3, enquanto a palavra não for descoberta ou se o número máximo de erros possíveis for atingido;
6. O sistema calcula a pontuação atingida pelo usuário;
7. A rodada é encerrada informando o resultado ao jogador;

### Fluxo Alternativo

Não há

### Pós condições

Após a execução deste caso do uso, as estatísticas de vitórias do jogador devem estar atualizadas de acordo com o resultado desta rodada.

## Regras de Negócio

1. O número máximo de erros que o jogador pode cometer é de 5 erros;
2. Caso o jogador repita um palpite errado que já foi realizado, o mesmo NÃO conta um novo erro;
3. A pontuação da rodada será calculada pelo número de letras que a palavra tem, menos os erros cometidos pelo jogador, não sendo admitido pontuações negativas;
